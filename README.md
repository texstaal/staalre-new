# STAAL Real Estate

Marketing site for STAAL Real Estate — boutique warehouse & logistics real estate
advisory in the Netherlands, acting for occupiers (tenants/buyers).

## Static site

The deployable site is plain static files served from the repo root:

- `index.html` — the page
- `css/`, `fonts/`, `images/`, `js/` — assets
- `vercel.json` — tells Vercel to serve the root with no build step

No build is required to deploy; Vercel serves the files as-is.

## Pages

- `/` (index.html) — homepage with the hero scroll sequence
- `/services`, `/process`, `/sectors`, `/partners`, `/about`, `/contact`
- `/insights` + three articles under `/insights/<slug>`
- `/terms`, `/privacy`, `/cookie-policy`, `/disclaimer`, `404.html`
- `sitemap.xml`, `robots.txt`; clean URLs via `vercel.json` (`cleanUrls: true`)

## Regenerating the site

`index.html` and `css/staal.css` are generated by `assemble.js`; all inner pages
are generated by `pages.js` (which extracts the header/footer from the
assembled homepage — so always run both, in order):

```
node assemble.js && node pages.js
```

Forms (contact + newsletter) post to Supabase via `js/forms.js`; until the
Supabase keys are filled in there, they fall back to a pre-addressed email.

- `gen-staal-logo.js` → `staal-logo.json` regenerates the STAAL letterform SVG
  paths from `InstrumentSans-var.ttf` (only needed if the wordmark changes).
- Hero scroll choreography, header, burger menu, testimonials carousel, reveals
  live in `js/main.js` (vanilla JS + local Swiper and Lenis).
